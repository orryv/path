About the different path formats:
- **AccessPath** = OS-native string for local FS access (\ on Windows; / on POSIX). Backslashes on Linux are literal characters, not separators. Uses Orryv\XString::encodeSafePath() for encoding non URIs, and withEncoding() method to set encoding typefor URIs.
- **ReferencePath** = portable, slash-only logical path for storage/compare/logging. Never percent-encode. Never \. Use UTF-8.
- **AccessURI** = a proper RFC 3986/8089 URI (file://, http://, …). Path segments percent-encoded with rawurlencode; query encoded with urlencode. Decode on the way back.

About encoding/decoding:
- For `OS Paths`:
  - With `AccessPath` input format:
    - To `ReferencePath`: uses `XString::decodeSafePath()` internally by default. use `withOSEncoding(PathFormat::ACCESS_PATH, PathFormat::REFERENCE_PATH, PathEncoding::DECODE_SAFE_PATH)`
    - To `AccessURI`: uses `rawurlencode()` internally by default.
  - With `ReferencePath` input format:
    - To `AccessPath`: uses `XString::encodeSafePath()` internally by default.
    - To `AccessURI`: uses `rawurlencode()` internally by default.
  - With `AccessURI` input format:
    - To `AccessPath`: uses `rawurldecode()` and `XString::decodeSafePath()` internally by default.
    - To `ReferencePath`: uses `rawurldecode()` internally by default.
- For `URIs`:
  - With `AccessPath` input format:
    - To `ReferencePath`: uses `urldecode()` internally by default.
    - To `AccessURI`: Identical to `AccessPath`.
  - With `ReferencePath` input format:
    - To `AccessPath`: uses `urlencode()` internally by default. (always urlencode on fragments and query parts?)
    - To `AccessURI`: uses `urlencode()` internally by default. (always urlencode on fragments and query parts?)
  - With `AccessURI` input format:
    - To `AccessPath`: Identical to `AccessURI`.
    - To `ReferencePath`: uses `urldecode()` internally by default.

Example input conversion (standard ACCESS_PATH input):
```
C:\Program Files\App\file.txt
  AccessPath     => C:\Program Files\App\file.txt
  ReferencePath  => C:/Program Files/App/file.txt
  AccessURI      => file:///C:/Program%20Files/App/file.txt

\\server\share\folder\sp ace.txt
  AccessPath     => \\server\share\folder\sp ace.txt
  ReferencePath  => //server/share/folder/sp ace.txt
  AccessURI      => file://server/share/folder/sp%20ace.txt

/var/www/html/index.html
  AccessPath     => /var/www/html/index.html
  ReferencePath  => /var/www/html/index.html
  AccessURI      => file:///var/www/html/index.html

https://ex.com/a b/c?x=1 y#ä
  UrlPath::toString(AccessURI)     => https://ex.com/a%20b/c?x=1%20y#%C3%A4
  UrlPath::toString(ReferencePath) => https://ex.com/a b/c?x=1 y#ä   (no re-encoding in path; query/fragment preserved as text)
```

## Files and Directories usage

`Path::absolute()` must accept:
  - Windows drive paths: `C:/projects/demo/file.txt`
  - Unix/Posix absolute paths: `/projects/demo/file.txt`
  - Windows UNC paths: `//server/share/folder/file.txt`
  - Windows Extended-length paths: `\\?\C:\projects\demo\file.txt` or `\\?\UNC\server\share\folder\file.txt`
  - URIs: `file:///C:/...`, `http://...`, `ftp://...`, ...

```php
// TODO: check that we are correct that there is no way to figure out if a path is a file or directory when we don't every call the system?

// All methods are immutable, meaning they return new instances and do not modify the current instance.
use Orryv\Path;
use Orryv\Path\Enums\PathFormat

# Factory (returns FilePath, DirectoryPath or UrlPath instance)
$path = Path::file('C:/projects/demo/file.txt', $input_format = PathFormat::ACCESS_PATH); // Returns FilePath instance
// $path = Path::dir('C:/projects/demo/', $input_format = PathFormat::ACCESS_PATH); // trailing slash optional
// $path = Path::dot('C:/projects/demo/file.txt', $input_format = PathFormat::ACCESS_PATH); // will detect if file or directory when it doesn't end with a slash, or when last part has a dot (.) in it.
// $path = Path::system('C:/projects/demo/file.txt', $input_format = PathFormat::ACCESS_PATH); // will detect if file or directory by making a system call (URI that is not file will throw exception)
// $path = Path::url('http://example.com/path/to/resource', $input_format = PathFormat::ACCESS_PATH); // specialized for URLs

# Options on FilePath, DirectoryPath or UrlPath
$path->withWindowsLongPathSupport($enabled = true); // true means will support Windows long paths (\\?\ prefix). default is false.
$path->withPreserveEndSlash($value = true); // true means directory paths will always end with a slash, false means they never will (except for root directories). default is true.



# Navigation
$path->setBaseDir(FilePath|DirectoryPath|UrlPath|string Path::dir('C:/projects/')); // Returns DirectoryPath or UrlPath instance Set base path for relative paths. if string will internally convert to DirectoryPath or UrlPath (depending on current instance). FilePath automatically uses parent directory as base.
$path->cd('..'); // Returns DirectoryPath or UrlPath instance. Used to prevent going above a certain folder. If setBaseDir() was used, "/" will redirect to base dir. else will go to the root directory.
$path->getRelativePathFrom(FilePath|DirectoryPath|UrlPath|string $from_path, $format = PathFormat::ACCESS_PATH); // Returns a relative path string. 
// Example: $path: C:/projects/demo/file.txt <- getRelativePathFrom('C:/projects/') = demo/file.txt
$path->getRelativePathTo(FilePath|DirectoryPath|UrlPath|string $to_path, $format = PathFormat::ACCESS_PATH); // Returns a relative path string. 
// Example: $path: C:/projects/demo/file.txt -> getRelativePathTo('C:/projects/demo/x/video.mp4') = x/video.mp4
$path->getCommonBasePath(FilePath|DirectoryPath|UrlPath|string $other_path, $format = PathFormat::ACCESS_PATH); // Returns common base path as FilePath, DirectoryPath or UrlPath instance.
// Example: $path: C:/projects/demo/file.txt <- getCommonBasePath('C:/projects/x/video.mp4') = C:/projects/

# FilePath, DirectoryPath or UrlPath methods
$path->toString($format = PathFormat::ACCESS_PATH);


# Manipulations (on FilePath, DirectoryPath or UrlPath instances)
withHost($host); // Returns new instance with changed host.
withScheme($scheme); // Returns new instance with changed scheme.
withQuery(string|array $query); // Returns new instance with changed query. Can be null to remove query.
withoutQuery(?string $key = null); // Returns new instance without query.
withFragment($fragment); // Returns new instance with changed fragment. Can be null to remove fragment.
withoutFragment(); // Returns new instance without fragment.
withDirectory($directory); // All types. Returns new instance with changed path directory. (C:/projects/demo/, /var/www/html/, //server/share/folder/ or /path/to/resource/)
withRoot($root); // All types. Returns new instance with changed path root. (C:/, /, //server/share/, file:/// or http://example.com/)
withFilename($filename); // FilePath only. Returns new FilePath instance with changed filename.
withBasename($basename); // FilePath only. Returns new FilePath instance with changed basename
withExtension($extension); // FilePath only. Returns new FilePath instance with changed extension.
withDirname($dirname); // FilePath only. Returns new DirectoryPath instance with changed dirname.
withUsername($username); // Returns new instance with changed username.
withPassword($password); // Returns new instance with changed password.
withPort($port); // Returns new instance with changed port.

// withUrlEncoding(Encoder::URLENCODE) // All types. Options: Encoder::DEFAULT, Encoder::NONE, Encoder::URLENCODE, Encoder::RAWURLENCODE. When UrlPath default is Encoder::URLENCODE, others Encoder::NONE. (applies when an ReferencePath is passed in Path and now requesting AccessPath or AccessURI output, or will decode when vice versa)
withOSEncoding($from_format, $to_format, ...$encoding_type); // All types. Options (executed in order):
// - PathEncoding::ENCODE_RAWURL, = rawurlencode
// - PathEncoding::DECODE_RAWURL, = rawurldecode
// - PathEncoding::ENCODE_URL, = urlencode
// - PathEncoding::DECODE_URL, = urldecode
// - PathEncoding::ENCODE_SAFE_PATH, = XString::encodeSafePath()
// - PathEncoding::DECODE_SAFE_PATH, = XString::decodeSafePath()
// - PathEncoding::NONE. 

# Retrieval (returns strings, arrays, ints, bools)
// $from: false means from start, 0 means from first folder after root.
// $end: false means to end, 0 means to last folder before filename (for FilePath) or last folder (for DirectoryPath), -1 means to second last folder, etc. positive numbers means counting from $end.
// Examples:
// Given path: C:/projects/demo/file.txt
// Full path retrievals:
// from = false, end = false → C:/projects/demo/file.txt
// from = false, end = -1 → C:/projects/demo/ (depends on preserveEndSlash setting if it ends with slash or not)
// from = 0, end = false → /projects/demo/file.txt
// from = 0, end = 0 → /projects/demo/
// from = 1, end = -1 → throws exception (no folders selected)
// $as_array: means it will be returned as an array (ex.: ['C:', 'projects', 'demo', 'file.txt']) instead of a string.
// or for url: ['http://example.com', 'path', 'to', 'resource']
$access_path = $path->getAccessPath(false|int $from = false, false|int $end = false, bool $as_array = false);      // C:\projects\demo\file.txt
$reference_path = $path->getReferencePath(false|int $from = false, false|int $end = false, bool $as_array = false);  // C:/projects/demo/file.txt
$access_uri = $path->getAccessURI(false|int $from = false, false|int $end = false, bool $as_array = false);      // file:///C:/projects/demo/file.txt
// Above 3 are the same as:
getPath($format = PathFormat::ACCESS_PATH, false|int $from = false, false|int $end = false, bool $as_array = false); // type: ACCESS_PATH_TYPE_ACCESS_PATH, ACCESS_PATH_TYPE_REFERENCE_PATH, ACCESS_PATH_TYPE_ACCESS_URI.
getRoot($format = PathFormat::ACCESS_PATH); // Returns path root as string. E.g. C:/, /, //server/share/, file:/// or http://example.com/
getDirectory($format = PathFormat::ACCESS_PATH); // Returns path directory. E.g. C:/projects/demo/ for file C:/projects/demo/file.txt
getFilename($format = PathFormat::ACCESS_PATH); // FilePath only. Returns filename as string. E.g. file.txt
getBasename($format = PathFormat::ACCESS_PATH); // FilePath only. Returns basename as string. E.g. file
getExtension($format = PathFormat::ACCESS_PATH); // FilePath only. Returns extension as string. E.g. txt
getFolderLevelCount(); // Returns int number of folder levels in path.

// Ideas
isWindowsPath()
isUnixPath()
isUNCPath()
isExtendedLengthPath()
isAbsolute()
isFilePath()
isDirectoryPath()
isUrlPath()
hasTrailingSlash()
```


## URL support:

When using Path::url(), the path will be resolved per RFC 3986 standard (change href values with our cd() method):

```
base = https://example.com/a/b/c/index.html?x=1#frag
href="" → https://example.com/a/b/c/index.html?x=1 (same doc; drops the base’s fragment)
href="." or "./" → https://example.com/a/b/c/
href=".." → https://example.com/a/b/
href="../d" → https://example.com/a/b/d
href="/" → https://example.com/
href="//cdn.example.com/x" → https://cdn.example.com/x (scheme inherits)
href="?y=2" → https://example.com/a/b/c/index.html?y=2
href="#top" → https://example.com/a/b/c/index.html?x=1#top
```

```php
use Orryv\Path;

// Resolving hrefs is easy:
Path::url($current_url)->cd($href)->toString(); // Going above root will throw exception OutOfBoundsException
```
