## Files and Directories usage

`Path::absolute()` must accept:
  - Windows drive paths: `C:/projects/demo/file.txt`
  - Unix/Posix absolute paths: `/projects/demo/file.txt`
  - Windows UNC paths: `//server/share/folder/file.txt`
  - Windows Extended-length paths: `\\?\C:\projects\demo\file.txt` or `\\?\UNC\server\share\folder\file.txt`
  - URIs: `file:///C:/...`, `http://...`, `ftp://...`, ...

```php
// TODO: check that we are correct that there is no way to figure out if a path is a file or directory when we don't every call the system?

// All methods are immutable, meaning they return new instances and do not modify the current instance.
use Orryv\Path;

# Factory (returns FilePath, DirectoryPath or UrlPath instance)
$path = Path::file('C:/projects/demo/file.txt');
// $path = Path::dir('C:/projects/demo/'); // trailing slash optional
// $path = Path::dot('C:/projects/demo/file.txt'); // will detect if file or directory based on trailing dot or slash
// $path = Path::system('C:/projects/demo/file.txt'); // will detect if file or directory by making a system call (URI that is not file will throw exception)
// $path = Path::url('http://example.com/path/to/resource'); // specialized for URLs

# Retrieval (returns AccessPath, ReferencePath or AccessURI instance)
#  $windows_long_path = true means it will return long path if it is Windows and exceeds MAX_PATH
$access_path = $path->getAccessPath($windows_long_path = false);      // C:\projects\demo\file.txt
$reference_path = $path->getReferencePath($windows_long_path = false);  // C:/projects/demo/file.txt
$access_uri = $path->getAccessURI($windows_long_path = false);      // file:///C:/projects/demo/file.txt
// Windows specific

# Output options (called on FilePath, DirectoryPath, UrlPath, AccessPath, ReferencePath or AccessURI instances)
$path->preserveEndSlash($value = true); // true means directory paths will always end with a slash, false means they never will (except for root directories). default is true.

# Navigation
$path->setBaseDir(FilePath|DirectoryPath|UrlPath|string Path::dir('C:/projects/')); // Returns DirectoryPath or UrlPath instance Set base path for relative paths. if string will internally convert to DirectoryPath or UrlPath (depending on current instance). FilePath automatically uses parent directory as base.
$path->cd('..'); // Returns DirectoryPath or UrlPath instance. Used to prevent going above a certain folder. If setBaseDir() was used, "/" will redirect to base dir. else will go to the root directory.
$path->getRelativePathFrom(FilePath|DirectoryPath|UrlPath|string $from_path, $format = ACCESS_PATH_FORMAT); // Returns a relative path string. 
// Example: $path: C:/projects/demo/file.txt <- getRelativePathFrom('C:/projects/') = demo/file.txt
$path->getRelativePathTo(FilePath|DirectoryPath|UrlPath|string $to_path, $format = ACCESS_PATH_FORMAT); // Returns a relative path string. 
// Example: $path: C:/projects/demo/file.txt -> getRelativePathTo('C:/projects/demo/x/video.mp4') = x/video.mp4
$path->getCommonBasePath(FilePath|DirectoryPath|UrlPath|string $other_path, $format = ACCESS_PATH_FORMAT); // Returns common base path as FilePath, DirectoryPath or UrlPath instance.
// Example: $path: C:/projects/demo/file.txt <- getCommonBasePath('C:/projects/x/video.mp4') = C:/projects/

# FilePath, DirectoryPath or UrlPath methods
$path->toString($format = ACCESS_PATH_FORMAT); // Can't return windows long paths.
# AccessPath, ReferencePath or AccessURI methods
$access_path->toString();


# Manipulations (on FilePath, DirectoryPath or UrlPath instances)
withHost($host); // UrlPath only. Returns new UrlPath instance with changed host.
withScheme($scheme); // UrlPath only. Returns new UrlPath instance with changed scheme.
withQuery($query); // UrlPath only. Returns new UrlPath instance with changed query.
withFragment($fragment); // UrlPath only. Returns new UrlPath instance with changed fragment.
withFilename($filename); // FilePath only. Returns new FilePath instance with changed filename.
withBasename($basename); // FilePath only. Returns new FilePath instance with changed basename
withExtension($extension); // FilePath only. Returns new FilePath instance with changed extension.
withDirname($dirname); // FilePath only. Returns new DirectoryPath instance with changed dirname.

# Querying 
getPathRoot($format = ACCESS_PATH_FORMAT); // Returns path root as string. E.g. C:/, /, //server/share/, file:/// or http://example.com/
getPathDirectory($format = ACCESS_PATH_FORMAT); // Returns path directory as DirectoryPath instance. E.g. C:/projects/demo/ for file C:/projects/demo/file.txt
getPathFilename($format = ACCESS_PATH_FORMAT); // FilePath only. Returns filename as string. E.g. file.txt
getPathBasename($format = ACCESS_PATH_FORMAT); // FilePath only. Returns basename as string. E.g. file
getPathExtension($format = ACCESS_PATH_FORMAT); // FilePath only. Returns extension as string. E.g. txt
```


## URL support:

When using Path::url(), the path will be resolved per RFC 3986 standard (change href values with our cd() method):

```
base = https://example.com/a/b/c/index.html?x=1#frag
href="" → https://example.com/a/b/c/index.html?x=1 (same doc; drops the base’s fragment)
href="." or "./" → https://example.com/a/b/c/
href=".." → https://example.com/a/b/
href="../d" → https://example.com/a/b/d
href="/" → https://example.com/
href="//cdn.example.com/x" → https://cdn.example.com/x (scheme inherits)
href="?y=2" → https://example.com/a/b/c/index.html?y=2
href="#top" → https://example.com/a/b/c/index.html?x=1#top
```

```php
use Orryv\Path;

// Resolving hrefs is easy:
Path::url($current_url)->cd($href)->toString(); // Going above root will throw exception OutOfBoundsException
```
